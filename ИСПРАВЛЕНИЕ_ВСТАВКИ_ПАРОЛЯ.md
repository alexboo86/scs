# Исправление вставки пароля в терминале Cursor

## Проблема

В терминале Cursor не работает вставка пароля через:
- Ctrl+V
- Shift+Insert  
- Правый клик мыши

Это происходит потому что `Read-Host -AsSecureString` блокирует вставку.

## Решение

Скрипты обновлены для использования обычного `Read-Host` вместо `Read-Host -AsSecureString`.

Теперь вы можете:
- ✅ Вставлять пароль через **Ctrl+V**
- ✅ Вставлять пароль через **Shift+Insert**
- ✅ Вставлять пароль через **правый клик мыши**

## Использование

### 1. Деплой с паролем:

```powershell
.\deploy-with-password.ps1
```

Когда появится запрос пароля:
1. Скопируйте пароль (Ctrl+C)
2. Вставьте через Ctrl+V или правый клик
3. Нажмите Enter

### 2. Настройка SSH ключей:

```powershell
.\setup-ssh-keys.ps1
```

Когда появится запрос пароля:
1. Скопируйте пароль (Ctrl+C)
2. Вставьте через Ctrl+V или правый клик
3. Нажмите Enter

## Важно

⚠️ **Безопасность**: Пароль теперь вводится как обычный текст (не скрыт звездочками), но это необходимо для возможности вставки.

После настройки SSH ключей пароль больше не потребуется!
